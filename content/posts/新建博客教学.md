# 1. y9000p 上如何上传文章

我建立了 Learn / Build / Think / Live 的四分 categories。

```
                Professional（专业）
                     ↑
                     │
   (Learn × Pro)     │     (Think × Pro)
     Learn           │      Think
   ----------------------------------------→
                     │
   (Learn × Life)    │     (Think × Life)
  Life-Learn         │      Live
                     │
                 Life（生活）
```
```
Learn：专业学习、知识输入
Build：项目与工程实践（不直接放入坐标，但属于专业输出）
Think：专业层面的思考、方法论、反思
Live：生活、随笔、个人体验
```

以后迁移博客的话，需要去问 AI，人的精力是有限的，但是 AI 的精力是无限的，在此简单陈述下怎么去在这台 y9000p 上新建博客，免得每次都要问 AI。

而且其中一些独特的东西，时间久了 AI 也不一定记得。

## 新建博客流程

首先在 VS Code 的 content/posts 中新建一个 md，然后打一个 hugo，并且 Ctrl + Space 完成 head 的建立。

接下来写自己想写的文章，写好后可以让 AI 调一下 markdown 格式，说起来这真的是我第一次在工作流或者说学习流中使用 AI。

## 发布文章

写好后完成了这个 md，可以：

```
git add .
git commit
git push
```

三步走，稍等一会就能看到文章了。

# 2. 怎么迁移博客

我也不能用一辈子 y9000p，所以还是得先说好迁移的事情。

换电脑 = 重新 git clone + 安装 Hugo + 推送即可。

## Step 1：安装最基础的 3 个工具

- Git
- Hugo（extended）
- VS Code（可选，但推荐）

不需要配置 Pages，不需要碰 GitHub 设置。

## Step 2：把博客仓库 clone 下来

```
git clone https://github.com/ichildhood/ichildhood.github.io.git
cd ichildhood.github.io
```

你用的是 Stack 主题 submodule：

```
git submodule update --init --recursive
```

## Step 3：验证本地是否能跑（可选但推荐）

```
hugo server
```

## Step 4：继续正常写博客

写 md 文件，然后：

```
git add .
git commit -m "post: new article"
git push origin master
```

希望以后能多多写博客。
